Documentación general

  • Obtener los datos de un cliente.

    • Para obtener los datos de un cliente especìfico, simplemente se llama a la URL
      /rest/cliente/{id_del_cliente}
    • Todos los clientes registrados con sus datos:
      /rest/cliente/

  • Listar todas las cuentas asociadas a un cliente.
    • Para obtener los datos de las cuentas de un cliente en especìfico, simplemente se llama a la URL
      /rest/cuenta/{id_del_cliente}
    • Todos las cuentas registradas con sus datos:
      /rest/cuenta/

  • Obtener los últimos 10 movimientos de una cuenta.
    /rest/movimiento/ultimos/{id_cuenta}
    • Donde {id_cuenta} representa el id de la cuenta de cual queremos conocer sus últimos 10 movimientos.

  • Crear un movimiento de cuenta.

  • Se hace mediante un JSON con la solicitud POST de la forma:
    • { "apertura": "{fecha de apertura}", "id": {id de cuenta}, "idCliente": {id de cliente}, "numero": {numero de cuenta} }

  • Actualizar un movimiento de cuenta.

  • Se hace mediante un JSON con la solicitud PUT de la forma:
    • { "apertura": "{fecha de apertura}", "id": {id de cuenta}, "idCliente": {id de cliente}, "numero": {numero de cuenta} }

  • Obtener el saldo de una cuenta.

    /rest/movimiento/{id_cuenta}/descripcion/{detalle_de_la_descripcion}
    • Donde {id_cuenta} representa el id de la cuenta de la cual queremos encontrar los movimientos, y {detalle_de_la_descripcion} es la descripción que queremos buscar.

  • Obtener todos los movimientos de una cuenta en un estado específico.
    /rest/movimiento/{id_cuenta}/{estado}
    • Donde {id_cuenta} representa el id de la cuenta de la cual queremos encontrar los movimientos, y {estado} el estado que queremos buscar

  • Buscar movimientos de una cuenta según una descripción dada.
    /rest/movimiento/{id_cuenta}/descripcion/{detalle_de_la_descripcion}
    • Donde {id_cuenta} representa el id de la cuenta de la cual queremos encontrar los movimientos, y {detalle_de_la_descripcion} es la descripción que queremos buscar.
